What each row means 
1. Protocol — getUserMedia requires HTTPS. http:// or file:// won't work.
2. mediaDevices — must exist on `navigator`. Some old browsers / iframes don't expose it.
3. Permission state — "granted" = ready · "prompt" = will show dialog · "denied" = blocked.
4. enumerateDevices — must list ≥1 device with kind "videoinput".
5. getUserMedia — actually requests stream. If permission was prompt, this fires the dialog.
   If permission denied, this throws NotAllowedError. If no device, NotFoundError.
   If in use, NotReadableError.
